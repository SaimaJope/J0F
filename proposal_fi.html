1. Yhteenveto

JobFuture Oy tarvitsee modernin, suomenkielisen tukkukaupan verkkokaupan LVI- ja kylmätuotteille. Sivusto toimii B2B-tuotekatalogina ostoskoritoiminnallisuudella, jossa kassatoiminto tuottaa PDF-tilauslomakkeen verkkomaksamisen sijaan. Asiakkaat lähettävät tilauksensa sähköpostitse luodun PDF:n avulla osoitteeseen info@jobfuture.fi.

Tuotetiedot haetaan kahdelta tavarantoimittajan verkkosivustolta — Ahlsell (INNOVA-tuotteet) ja Onninen (kylmätuotekategoriat) — molempien toimittajien luvalla. Tiedot järjestetään asianmukaisiin alakategorioihin ja valmistajakohtaisiin ryhmiin JobFuturen omassa katalogissa.

Keskeinen arvolupaus: Ammattimainen ja helppokäyttöinen digitaalinen katalogi, joka korvaa manuaaliset hinnastot ja puhelintilaukset. Ratkaisu tehostaa B2B-tilausprosessia säilyttäen samalla henkilökohtaisen palvelun sähköpostipohjaisen tilausvahvistuksen kautta.

2. Projektin laajuus ja tavoitteet

Sisältyy projektiin

  • Suomenkielinen responsiivinen verkkosivusto (työpöytä + mobiili)
  • Tuotekatalogi kategorioilla, alakategorioilla ja valmistajasuodatuksella
  • Tuotehaku automaattitäydennyksellä
  • Ostoskori määränhallinnalla
  • PDF-tilauslomakkeen luonti kassalla (ei verkkomaksamista)
  • Tuotetietojen tuonti Ahlsellista (INNOVA) ja Onnisesta (6 kategoriaa) — toimittajien luvalla
  • Hallintapaneeli tuotteiden hallintaan, hinnoitteluun ja katalogin päivitykseen
  • Yhteystiedot ja yrityssivut

Ei sisälly 1. vaiheeseen

  • Verkkomaksaminen
  • Käyttäjärekisteröinti / asiakastilit
  • Automaattinen tilaustenkäsittely / ERP-integraatio
  • Monikielisyys (vain suomi tässä vaiheessa)

5. Teknologiavalinta

Suositeltu teknologiapino, joka on optimoitu katalogi-/tukkukauppasivustolle ja PDF-luontitarpeisiin.

5.1 Käyttöliittymä (Frontend)

Next.js 15 React 19 TypeScript Tailwind CSS

  • Next.js — Full-stack React-kehys palvelinpuolen renderöinnillä nopeisiin sivulatauksiin ja hakukoneoptimointiin
  • Tailwind CSS — Utility-first CSS nopeaan ja yhtenäiseen käyttöliittymäkehitykseen
  • TypeScript — Tyyppitarkistus koko koodikannan laajuudelta

5.2 Palvelinpuoli ja API

Next.js API Routes Prisma ORM PostgreSQL

  • Next.js API Routes — Palvelinlogiikka samassa projektissa käyttöliittymän kanssa, mikä yksinkertaistaa käyttöönottoa
  • Prisma — Tyyppiturvallinen ORM tietokantaoperaatioihin migraatiotuella
  • PostgreSQL — Luotettava relaatiotietokanta rakenteelliselle tuotetiedolle

5.3 Tiedonkeruu

Python Playwright BeautifulSoup

  • Python + Playwright — Selainautomaatio kirjautuneille tiedonkeruuistunnoille
  • BeautifulSoup — HTML-jäsennys ja tietojen poiminta
  • Tiedonkeruuskriptit ajetaan itsenäisesti; tulokset tuodaan PostgreSQL-tietokantaan

5.4 PDF-luonti

React-PDF tai Puppeteer

  • Palvelinpuolen PDF-luonti tilausdatasta
  • Brändätty pohja JobFuturen logolla, asiakastiedoilla, tuotetaulukolla ja loppusummilla

5.5 Palvelinympäristö ja infrastruktuuri

Render PostgreSQL

  • Render: Kaikki yhdellä alustalla — Next.js-hostaus ja hallittu PostgreSQL. Yksinkertainen, kustannustehokas ja kaikki yhdessä hallintapaneelissa.
  • Automaattinen käyttöönotto Gitistä, sisäänrakennettu SSL ja EU-alueen tuki
Alusta: Render — yksi alusta verkkosovellukselle, tietokannalle ja staattisille tiedostoille. Minimoi kustannukset ja operatiivisen monimutkaisuuden.

6. Ulkoasu ja käyttökokemus

6.1 Suunnitteluperiaatteet

  • Ammattimainen ja selkeä: B2B-estetiikka — ei sekavuutta, selkeä tietohierarkia
  • Nopea ja toimiva: Tukkukaupan ostajat arvostavat tehokkuutta, eivät näyttäviä animaatioita
  • Mobiiliresponsiivinen: Asentajat saattavat selata sivustoa työmailla puhelimilla/tableteilla

7. PDF-tilauslomakkeen luonti ja työnkulku

7.1 Tilauksen kulku

  1. Asiakas selaa katalogia ja lisää tuotteita ostoskoriin
  2. Asiakas klikkaa "Tee tilaus"
  3. Asiakas täyttää yrityksen ja yhteyshenkilön tiedot tilauslomakkeelle
  4. Järjestelmä luo brändätyn PDF:n, joka sisältää:
    • JobFuturen logo ja yritystiedot
    • Asiakkaan/yrityksen tiedot
    • Tilauspäivämäärä ja yksilöllinen viitenumero
    • Tuotetaulukko: tuotekoodi, nimi, määrä, yksikköhinta, rivihinta
    • Välisumma, ALV (25,5 %) ja loppusumma
    • Huomautuskenttä
  5. PDF ladataan asiakkaan laitteelle
  6. Asiakkaalle näytetään ohje: "Lähetä tilaus sähköpostilla osoitteeseen info@jobfuture.fi"
  7. Valinnainen lisäominaisuus: "Lähetä tilaus" -painike, joka avaa valmiiksi täytetyn sähköpostiluonnoksen PDF-liitteellä

7.2 PDF-pohjan rakenne

OsioSisältö
YläosaJobFuturen logo, yrityksen nimi, osoite, Y-tunnus
AsiakastiedotYrityksen nimi, yhteyshenkilö, sähköposti, puhelin, toimitusosoite
Tilauksen metatiedotTilauksen viitenumero, päivämäärä, maksuehdot
TuotetaulukkoSarakkeet: #, tuotekoodi, tuotenimi, määrä, yksikköhinta, summa
YhteenvetoVälisumma (alv 0 %), ALV 25,5 %, loppusumma
AlaosaHuomautukset, "Lähetä osoitteeseen info@jobfuture.fi", yrityksen alatunniste

8. Projektin vaiheet ja aikataulu

Vaihe 1 — Perusta ja tiedonkeruu (Viikot 1–2)
  • Projektin repositorion perustaminen, Next.js-runko, tietokantaskeema
  • Python-tiedonkeruuskriptien rakentaminen Ahlsellille (INNOVA) ja Onniselle
  • Kirjautuneiden istuntojen, sivutuksen ja tietojen poiminnan käsittely
  • Kerättyjen tuotteiden tuonti PostgreSQL-tietokantaan oikeilla kategorioilla
  • Tuotekuvien tallennuksen perustaminen
Vaihe 2 — Katalogi ja käyttöliittymä (Viikot 3–4)
  • Tuotelistaussivujen rakentaminen kategorianavigaatiolla
  • Tuotesivujen toteutus
  • Hakutoiminnallisuuden rakentaminen automaattitäydennyksellä
  • Suodattimien toteutus (kategoria, valmistaja, hintahaarukka)
  • Responsiivinen suunnittelu kaikille näyttöko'oille
Vaihe 3 — Ostoskori ja PDF-kassa (Viikot 5–6)
  • Ostoskorin rakentaminen lisäys-/poisto-/määränhallinnalla
  • Tilauslomakkeen luonti validoinnilla
  • PDF-luonnin toteutus brändätyllä pohjalla
  • "Lataa PDF" ja valinnaisen "lähetä tilaus" -toiminnon rakentaminen
  • Ostoskorin pysyvyys (localStorage + istunto)
Vaihe 4 — Hallintapaneeli ja viimeistely (Viikot 7–8)
  • Hallintapaneelin autentikointi (yksinkertainen ja turvallinen kirjautuminen)
  • Tuotteiden hallinta (lisäys, muokkaus, poisto, massatuonti)
  • Kategorianhallintaliittymä
  • Uudelleenkeruun käynnistys erojen tunnistuksella (uudet/päivitetyt/poistetut tuotteet)
  • Hinnoittelun ylikirjoitus (mahdollisuus asettaa omat JobFuture-hinnat)
Vaihe 5 — Testaus ja julkaisu (Viikot 9–10)
  • Selain- ja laitetestaus
  • Suorituskyvyn optimointi (kuvien laiska lataus, välimuisti)
  • Hakukoneoptimoinnin asetukset (metatiedot, sivukartta, rakenteellinen data)
  • Verkkotunnuksen asetukset ja DNS-konfiguraatio
  • Käyttöönotto tuotantoympäristössä
  • Luovutusdokumentaatio ja hallintapaneelin koulutus
Arvioitu kokonaisaikataulu: 8–10 viikkoa projektin aloituksesta julkaisuun. Aikataulu on joustava ja sitä voidaan mukauttaa palautekierrosten ja laajuuden tarkennusten perusteella.

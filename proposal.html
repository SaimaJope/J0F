1. Executive Summary

JobFuture Oy requires a modern, Finnish-language wholesale catalog website for HVAC and refrigeration products. The site will function as a B2B product catalog with cart functionality, where the checkout process generates a PDF order form rather than processing payments online. Customers submit orders by emailing the generated PDF to info@jobfuture.fi.

Product data will be sourced from two supplier websites — Ahlsell (INNOVA-branded products) and Onninen (refrigeration categories) — with full authorization from both suppliers. Data will be organized into proper subcategories and manufacturer groupings within the JobFuture catalog.

Key Value Proposition: A professional, easy-to-use digital catalog that replaces manual price lists and phone orders, streamlining the B2B ordering workflow while keeping the human touch of email-based order confirmation.

2. Project Scope & Objectives

In Scope

  • Finnish-language responsive website (desktop + mobile)
  • Product catalog with categories, subcategories, and manufacturer filtering
  • Product search with autocomplete
  • Shopping cart with quantity management
  • PDF order generation at checkout (no payment processing)
  • Product data import from Ahlsell (INNOVA) and Onninen (6 categories) — with supplier authorization
  • Admin panel for product management, pricing, and catalog updates
  • Contact information and company pages

Out of Scope (Phase 1)

  • Online payment processing
  • User registration / customer accounts
  • Automated order processing / ERP integration
  • Multi-language support (Finnish only for now)

Each scraped product will be stored with the following structure:

FieldTypeDescription
idUUIDInternal unique identifier
nameStringProduct name (Finnish)
skuStringProduct code / SKU from supplier
descriptionTextProduct description
priceDecimalUnit price (EUR, ex. VAT)
imagesArrayProduct image URLs (downloaded locally)
manufacturerStringBrand / manufacturer name
categoryFKLink to category
subcategoryFKLink to subcategory
sourceEnumahlsell | onninen
source_urlStringOriginal product page URL
specificationsJSONTechnical specs (key-value pairs)
last_scrapedDateTimeWhen the product was last updated from source

4.4 Category Structure

Products will be organized into a two-level hierarchy:

  • INNOVA Products (from Ahlsell) — sub-organized by product type / manufacturer subcategory
  • Venttiilit — Valves and valve accessories
  • Asennustarvikkeet — Refrigeration installation supplies
  • Kylmäaineet — Refrigerants and heat transfer fluids
  • Ohjaus ja valvonta — Control, monitoring, and automation
  • Asennuskanavat — Installation channels and cable trays
  • Suojakourut — Protective conduits and covers

Additional categories and subcategories can be created via the admin panel as needed.

7. PDF Order Generation & Workflow

7.1 Order Flow

  1. Customer browses catalog and adds products to cart
  2. Customer clicks "Tee tilaus" (Make Order)
  3. Customer fills in company/contact details on the order form
  4. System generates a branded PDF containing:
    • JobFuture logo and company details
    • Customer/company information
    • Order date and unique reference number
    • Product table: SKU, name, quantity, unit price, line total
    • Subtotal, VAT (25.5%), and grand total
    • Notes/comments field
  5. PDF is downloaded to customer's device
  6. Customer is shown instructions: "Lähetä tilaus sähköpostilla osoitteeseen info@jobfuture.fi"
  7. Optional enhancement: "Send Order" button that auto-attaches PDF to a pre-filled email draft

7.2 PDF Template Layout

SectionContent
HeaderJobFuture logo, company name, address, Y-tunnus (business ID)
Customer InfoCompany name, contact person, email, phone, delivery address
Order MetaOrder reference #, date, payment terms
Product TableColumns: #, SKU, Product Name, Qty, Unit Price, Total
SummarySubtotal (ex. VAT), VAT 25.5%, Grand Total
FooterNotes, "Lähetä osoitteeseen info@jobfuture.fi", company footer

8. Project Phases & Timeline

Phase 1 — Foundation & Data Import (Weeks 1–2)
  • Set up project repository, Next.js scaffold, database schema
  • Build Python scraping scripts for Ahlsell (INNOVA) and Onninen
  • Handle authenticated sessions, pagination, and data extraction
  • Import scraped products into PostgreSQL with proper categorization
  • Set up product image storage
Phase 2 — Catalog & Frontend (Weeks 3–4)
  • Build product listing pages with category navigation
  • Implement product detail pages
  • Build search functionality with autocomplete
  • Implement filters (category, manufacturer, price range)
  • Responsive design across all breakpoints
Phase 3 — Cart & PDF Checkout (Weeks 5–6)
  • Build shopping cart with add/remove/quantity management
  • Create order form with validation
  • Implement PDF generation with branded template
  • Build "download PDF" and optional "email order" flow
  • Cart persistence (localStorage + session)
Phase 4 — Admin Panel & Polish (Weeks 7–8)
  • Admin authentication (simple, secure login)
  • Product management (add, edit, delete, bulk import)
  • Category management interface
  • Re-scrape trigger with diff detection (new/updated/removed products)
  • Pricing override (ability to set custom JobFuture prices)
Phase 5 — Testing & Launch (Weeks 9–10)
  • Cross-browser and device testing
  • Performance optimization (image lazy loading, caching)
  • SEO setup (meta tags, sitemap, structured data)
  • Domain setup and DNS configuration
  • Deployment to production environment
  • Handover documentation and admin training
Total estimated timeline: 8–10 weeks from project kickoff to launch. Timeline is flexible and can be adjusted based on feedback cycles and scope refinements.

10. Risks & Considerations

RiskImpactMitigation
Scraping fragility Source sites may change HTML structure, breaking scrapers Modular scraper design with error alerting; periodic manual review; admin re-scrape trigger
IP blocking Supplier sites may block scraping requests Respectful rate limiting; rotating user agents; run from VPS IP; only scrape during off-peak hours
Price accuracy Scraped prices may become stale Scheduled re-scraping (weekly or daily); "prices subject to confirmation" disclaimer on site; admin override capability
Authentication changes Login flows on source sites may change (2FA, captcha) Monitor login success; fallback to manual CSV import; admin notification on scrape failure
Data Access Authorization: Permission has been obtained from both Ahlsell and Onninen for automated product data extraction, eliminating the primary legal risk. Regular communication with both suppliers is recommended to maintain this arrangement and explore potential direct data feed or API access as a long-term enhancement.
